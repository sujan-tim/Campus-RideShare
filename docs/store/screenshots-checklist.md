# Screenshot Checklist

Capture screenshots from release builds with final branding, not from the browser dev server.

## Apple App Store

Use screenshots to show the real in-app experience.

Checklist:

- Provide at least 1 screenshot and at most 10 per required device size
- Use `.png` or `.jpg`
- If the UI is the same across device sizes, Apple allows using the highest required resolution and scaling down
- Optional: add up to 3 app previews per device size and language

Suggested RUride screenshot set:

1. Onboarding or login screen
2. Home screen with live campus map
3. Ride booking flow
4. Active ride tracking with driver map
5. Rutgers bus tracking screen
6. Stop details with ETAs
7. Payment method or checkout screen
8. Ride history or profile screen

## Google Play

Checklist:

- Minimum 2 screenshots across device types to publish the listing
- Use JPEG or 24-bit PNG with no alpha
- Minimum dimension: 320px
- Maximum dimension: 3840px
- Max dimension cannot be more than twice the min dimension

Suggested Play assets:

- Phone screenshots first
- Optional tablet screenshots if you support tablet layouts well
- Feature graphic
- Optional promo video

## Capture Order For RUride

- Splash / launch
- Home map
- Ride booking
- Live trip
- Bus map
- ETAs and stop sheet
- Payments
- Profile / history

## Final QA Before Upload

- No placeholder text
- No debug banners
- No fake broken payment or error states
- No visible personal information
- Status bar battery/time looks clean
- Maps are loaded and not blank
